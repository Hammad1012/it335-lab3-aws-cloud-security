# IT335 – Lab 3: AWS Cloud Security Fundamentals

This lab applies lessons from the Capital One cloud breach by configuring 
foundational AWS security controls across identity, storage, compute, and monitoring.

## What's Covered
- IAM least-privilege user/group setup with MFA verification
- Private S3 bucket with Block all public access enabled
- EC2 instance with restricted Security Group and IMDSv2 enabled
- CloudTrail logging and Billing dashboard review

## Repo Structure
screenshots/before/ – baseline screenshots before any configuration
screenshots/after/  – screenshots after each security control was applied
lab3-reflection.md  – written reflection connecting configurations to the Capital One breach
